# 배틀필드 메커니즘

CROBOTS의 물리 규칙과 전투 메커니즘을 설명합니다.

---

## 전장 (Battlefield)

전장은 **1,000m × 1,000m** 정사각형입니다.

```
y=999 +---------------------------+
      |                           |
      |        전 장              |
      |                           |
      |                           |
      |                           |
  y=0 +---------------------------+
      x=0                     x=999
```

- 좌측 하단이 원점 `(0, 0)`, 우측 상단이 `(999, 999)`
- 전장 바깥에는 **벽**이 있으며 충돌 시 데미지를 받습니다

---

## 방향 (Heading) 시스템

CROBOTS는 **나침반과 반대 방향** 체계를 사용합니다.

```
         90° (북)
          |
180° -----+----- 0° (동)
(서)      |
         270° (남)
```

| 방향 | 각도 |
|------|------|
| 동 (East) | 0° |
| 북 (North) | 90° |
| 서 (West) | 180° |
| 남 (South) | 270° |
| 북동 (NE) | 45° |
| 북서 (NW) | 135° |

- 각도는 0 — 359 범위이며 음수는 자동으로 절댓값 처리됩니다
- 360° 이상은 자동으로 360으로 나눈 나머지로 처리됩니다

## 로봇 이동

### 속도와 가속도

`drive(degree, speed)` 호출로 목표 속도를 설정하지만, 즉시 반영되지 않습니다.

- 매 **모션 사이클**(15 CPU 사이클)마다 현재 속도가 목표 속도를 향해 **10%씩** 변합니다
- 최고 속도 100% 로봇이 전력 질주 시 1초(15 모션 사이클)에 약 105m 이동

**가속 예시:**

```
목표 속도: 100%
현재 속도: 0 → 10 → 20 → 30 → ... → 100  (10 모션 사이클 소요)
```

### 방향 전환

방향 전환은 **속도 50% 이하**일 때만 가능합니다.

```c
/* 잘못된 예: 속도 100%에서 방향 전환 시도 → 먼저 감속됨 */
drive(0, 100);    /* 동쪽으로 전력 질주 */
drive(90, 100);   /* 북쪽으로 전환 시도 → 속도 50%까지 자동 감속 후 전환 */
```

```c
/* 올바른 패턴: 감속 후 전환 */
drive(cur_heading, 0);   /* 먼저 정지 */
while (speed() > 50)     /* 50% 이하가 될 때까지 대기 */
  ;
drive(new_heading, 100); /* 새 방향으로 출발 */
```

### 벽 충돌

벽에 부딪히면:
- 속도가 즉시 **0**으로 감소
- **2%** 데미지
- `drive()`로 새 방향을 설정해야 이동 재개 가능

## 스캐너

- 스캐너는 즉시 결과를 반환합니다 (이동과 관계없이 동작)
- 스캔 방향은 로봇의 이동 방향과 **독립적**입니다 (독립 포탑처럼 동작)
- 최대 사거리 제한 없음 — 전장 어느 곳이든 스캔 가능

**해상도 선택 전략:**

| resolution | 커버 각도 | 용도 |
|:----------:|:---------:|------|
| 0 | 정확히 1도 | 정밀 추적 (적 위치를 이미 알 때) |
| 1 | ±1° = 3도 | 좁은 범위 추적 |
| 5 | ±5° = 11도 | 일반 스캔 |
| 10 | ±10° = 21도 | 넓은 범위 탐색 |

전체 360도를 `resolution=10`으로 커버하려면 최소 **18번** 스캔 필요:
```c
int angle;
angle = 0;
while (1) {
  if (scan(angle, 10) > 0) {
    /* 적 발견 */
  }
  angle += 20;   /* 20도씩 이동 (±10도 겹치지 않게) */
  angle %= 360;
}
```

## 캐논과 미사일

### 미사일 특성

| 항목 | 값 |
|------|----|
| 최대 사거리 | 700m |
| 미사일 속도 | 50m/모션 사이클 |
| 최대 동시 비행 수 | 2발 |
| 재장전 시간 | 15 모션 사이클 |

700m 거리 미사일의 비행 시간: `700 / 50 = 14 모션 사이클`

### 재장전 관리

재장전 중에는 `cannon()` 호출이 `0`을 반환합니다.

```c
/* 발사될 때까지 대기하는 패턴 */
while (cannon(angle, range) == 0)
  ;

/* 재장전을 기다리는 동안 계속 스캔 */
while (cannon(angle, range) == 0) {
  range = scan(angle, 2);   /* 재장전 중에도 추적 */
}
```

## 데미지 시스템

### 데미지 원인별 수치

| 원인 | 데미지 | 조건 |
|------|--------|------|
| 벽 충돌 | 2% | 벽에 부딪힐 때 |
| 로봇 충돌 | 2% | 다른 로봇과 충돌 (양쪽 모두 데미지) |
| 미사일 원거리 | 3% | 폭발 지점에서 40m 이내 |
| 미사일 근거리 | 5% | 폭발 지점에서 20m 이내 |
| 미사일 직격 | 10% | 폭발 지점에서 5m 이내 |

- 데미지는 **누적**되며 회복되지 않습니다
- 데미지 수치가 높아도 속도, 스캔, 발사 능력은 전혀 영향 없습니다
- **100%에 도달하면 즉시 파괴**됩니다

### 데미지 감지 패턴

```c
int d;
d = damage();

while (1) {
  /* ... 전투 루틴 ... */

  if (damage() != d) {    /* 데미지가 바뀌었다 = 맞았다 */
    d = damage();
    avoid();              /* 회피 기동 */
  }
}
```

## 시뮬레이션 사이클

CROBOTS의 시뮬레이션은 다음 순서로 진행됩니다:

```
CPU 사이클 루프 (각 로봇이 1 사이클씩 순서대로 실행)
│
├── 매 15 CPU 사이클마다: 모션 업데이트
│   ├── 모든 로봇 위치/속도/방향 업데이트
│   ├── 모든 미사일 위치 업데이트
│   └── 충돌/데미지/사망 처리
│
└── 매 30 CPU 사이클마다: 프레임 기록 (리플레이용)
```

**제한 조건:**
- 최대 CPU 사이클: **2,000,000** (배틀 타임아웃)
- 이 한도에 도달하면 데미지가 적은 쪽이 승리

---

## 승리 조건

1. **상대 로봇의 데미지가 100%에 도달** — 즉시 승리
2. **CPU 사이클 제한(2,000,000) 도달** — 데미지가 더 적은 쪽 승리
3. **동시에 모든 로봇 파괴** — 무승부
